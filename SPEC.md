# 《汽车之家车型口碑采集》skill 需求说明 v0.2

> 本文档为正式版需求说明，后续开发与迭代以此为准。

---

## 1. skill 基础信息

| 项目 | 内容 |
|---|---|
| **skill 名称** | 汽车之家车型口碑采集 |
| **使用场景** | 从汽车之家指定车型口碑页批量采集用户评价，输出结构化 Excel |
| **目标用户** | 需要快速获取某车型口碑数据、进行整理分析或沉淀口碑素材的人员 |

---

## 2. 输入

- **方案 A**：完整 URL
  - 例如：`https://k.autohome.com.cn/8208?dimensionid=10&order=0&yearid=0#listcontainer`
- **方案 B**：车型 ID + 维度参数 + 页码范围
  - 例如：`seriesId=8208, dimensionid=10, pages=1-13`
- **方案 C**：按自然语言指定
  - 例如：`抓取风云X3L最满意和最不满意 1-13 页，导出 Excel`
- **方案 D**：车型 ID / URL + 全量采集意图
  - 例如：`抓取 8208 所有页面口碑，导出 Excel`
  - 若用户未明确最后页，系统应先检测总页数，再执行抓取

---

## 3. 输出

**Excel 文件（.xlsx）**

默认命名规则：
- `ZJ+车型名称+最满意or最不满意_页数范围.xlsx`
- 例如：`ZJ风云X3L最满意or最不满意_1-13页.xlsx`
- 若未显式指定页数范围，可省略页数后缀

默认 sheet 规则（正式版）：
- `最满意_购车口碑`
- `最满意_试驾探店口碑`
- `最不满意_购车口碑`
- `最不满意_试驾探店口碑`

如用户明确要求简化，可退化为两个 sheet：
- `最满意`
- `最不满意`

| 11 | 参考价格 | 试驾探店口碑字段 |

## 4. URL 规则

### 4.1 维度参数

| dimensionid | 维度 |
|---|---|
| 10 | 最满意 |
| 11 | 最不满意 |

### 4.2 分页规则

| 页码 | URL 格式 |
|---|---|
| 第 1 页 | `https://k.autohome.com.cn/{seriesId}?dimensionid={dimensionid}&order=0&yearid=0#listcontainer` |
| 第 2 页 | `https://k.autohome.com.cn/{seriesId}/index_2.html?dimensionid={dimensionid}&order=0&yearid=0#listcontainer` |
| 第 n 页 | `https://k.autohome.com.cn/{seriesId}/index_{n}.html?dimensionid={dimensionid}&order=0&yearid=0#listcontainer` |

## 5. 字段提取规则

| 字段 | 提取方式 |
|---|---|
| 口碑维度 | 根据 URL 参数 `dimensionid` 判断：10→最满意，11→最不满意 |
| 数据类型 | 根据字段与正文特征判断：车主购车口碑 / 试驾探店口碑 |
| 用户名 | 从页面用户名区域提取，按用户名规范化规则处理 |
| 发表日期 | 从“发表口碑”日期字段提取 |
| 口碑标题 | 从标题区域提取原文 |
| 综合口碑 | 从评分展示区域提取数字，如 4.86 |
| 车型 | 从车型标签区域提取，如 2025款 215智行版 |
| 行驶里程 | 从列表项 `4500km` 格式提取 |
| 电耗 | 从列表项 `14.6kWh 春秋电耗` 格式提取，原文保留 |
| 裸车购买价 | 从 `13.39万 裸车购买价` 提取，仅用于购车口碑 |
| 参考价格 | 从 `15.99万 参考价格` 提取，仅用于试驾探店口碑 |
| 购买时间 | 从 `2025-10 购买时间` 提取 |
| 探店时间 | 从 `2025-10 探店时间` 提取 |
| 购买地点 | 从 `赤峰 购买地点` 提取 |
| 探店地点 | 从 `芜湖 探店地点` 提取 |
| 评价详情 | 从正文区域提取原始展示文本，不做任何清洗、摘要、纠错 |
| 来源链接 | 提取“查看完整口碑”详情页 URL |
| 抓取页码 | 记录当前分页编号 |

### 5.1 用户名规范化规则

- 不删除“认证”信息
- 若页面展示为：`真实用户名 风云X3L认证`
- 输出规范化为：`真实用户名_风云X3L认证`
- 普通 `真实用户名 风云X3L` 形式可按需保留原样或仅去掉车型尾缀，但不得改动用户名本体

### 5.2 数据类型判定规则

满足任一条件时，判为 `试驾探店口碑`：

- 出现 `参考价格`
- 出现 `探店时间`
- 出现 `探店地点`
- 正文前缀为 `好评` 或 `槽点`
- 标题或正文明确出现 `试驾` 或 `探店`

否则默认判为 `车主购车口碑`。

### 5.3 关键约束

- **评价详情字段**：必须严格保留页面原文，不做任何删改、摘要、语义清洗、错字纠正
- **字段边界**：如遇页面异常文本混入，应识别并排除在正文外；不得因文风口语化、像广告、像生成式内容就擅自删除正文
- **字段拆列**：正式版中价格 / 时间 / 地点字段必须按购车口径与试驾口径拆列，不再混列
- **导出分 sheet 规则**：正式版默认按“维度 × 数据类型”拆成 4 个 sheet
- **记录有效性校验**：每条记录至少应包含“用户名 / 综合口碑 / 车型 / 来源链接”四项核心字段；若缺失则标记为异常记录，不直接作为正式结果写入
- **分页失败重试**：单页抓取失败时自动重试 2 次；连续失败后记录错误并中断或跳过，避免静默漏页
- **详情页抽样验证**：正式开发前至少抽样对比 5 条记录，核对列表页正文与详情页正文是否一致；若详情页更完整，则后续改为“列表页取结构字段 + 详情页补正文全文”
- **详情页双段补齐**：若同一来源链接仅在“最满意”或“最不满意”单侧列表页出现，但详情页实际同时包含两段正文，则以详情页为最终真相源，补齐缺失侧后再参与配平与导出
- **空值处理**：如某字段页面无展示，则该单元格留空

## 6. 技术路线

### 6.1 主方案：页面文本提取

- 使用 headless browser 打开口碑列表页
- 从页面可见文本中提取结构字段
- 优先读取页面文本，不默认走 OCR

### 6.2 兜底方案

- 如关键字段因页面结构异常无法提取
- 可局部使用 OCR 辅助识别
- 但 OCR 不作为默认方案

### 6.3 分页遍历

- 若用户明确指定页码范围，则按指定范围遍历
- 若用户未指定最后页，或表达“全部页面 / 所有页面 / 所有条目”，则先自动检测分页总页数
- 自动检测可通过首页分页控件、尾页链接或分页文本（如“共 N 页”）取得
- 每页抓取完成后执行记录数与核心字段校验
- 异常页进入重试或人工复核

## 8. 暂缓纳入正式版的字段

以下字段暂不纳入 v0.2：

- 来源页 URL
- 点赞数 / 评论数 / 浏览数
- 油耗 / 续航

原因：
- 当前主要目标是稳定完成口碑结构化采集
- 以上字段可在 v0.3 作为扩展项加入
