# 《汽车之家车型口碑采集》skill 需求说明 v0.5

> 本文档为当前正式版需求说明，后续开发与迭代以此为准。

---

## 1. skill 基础信息

| 项目 | 内容 |
|---|---|
| **skill 名称** | 汽车之家车型口碑采集 |
| **使用场景** | 从汽车之家指定车型口碑页批量采集全部口碑，输出结构化 Excel |
| **目标用户** | 需要快速获取某车型口碑数据、进行整理分析或沉淀口碑素材的人员 |

---

## 2. 输入

- **方案 A**：完整 URL
  - 例如：`https://k.autohome.com.cn/8140?dimensionid=0&order=0&yearid=0#listcontainer`
- **方案 B**：车型 ID + 页码范围
  - 例如：`seriesId=8140, pages=1-13`
- **方案 C**：按自然语言指定
  - 例如：`抓取启源A06全部口碑 1-13 页，导出 Excel`
- **方案 D**：车型 ID / URL + 全量采集意图
  - 例如：`抓取 8140 所有页面口碑，导出 Excel`
  - 若用户未明确最后页，系统应先检测总页数，再执行抓取

---

## 3. 输出

**Excel 文件（.xlsx）**

默认命名规则：
- `ZJ+车型名称+口碑_页数范围.xlsx`
- 例如：`ZJ启源A06口碑_1-13页.xlsx`
- 若未显式指定页数范围，可省略页数后缀

默认 sheet 规则：
- `口碑`

### 3.1 字段清单

| 列号 | 字段名 | 说明 |
|---|---|---|
| 1 | 数据类型 | 车主购车口碑 / 试驾探店口碑 |

| 8 | 电耗 | 原文保留，如 `14.6kWh 春秋电耗` |
| 9 | 裸车购买价 | 车主购车口碑字段 |
| 10 | 参考价格 | 试驾探店口碑字段 |
| 11 | 购买时间 | 车主购车口碑字段 |

## 5. 字段提取规则

| 字段 | 提取方式 |
|---|---|
| 数据类型 | 根据字段与正文特征判断：车主购车口碑 / 试驾探店口碑 |
| 用户名 | 从列表页或详情页用户名区域提取，按用户名规范化规则处理 |
| 发表日期 | 从“发表口碑”日期字段提取 |
| 口碑标题 | 从标题区域提取原文 |
| 综合口碑 | 优先用详情页 7 个维度分数求平均；分项不完整时回退到列表页展示值 |
| 车型 | 从车型标签区域提取 |
| 行驶里程 | 从列表项 `4500km` 格式提取 |
| 电耗 | 从列表项 `14.6kWh 春秋电耗` 格式提取，原文保留 |
| 裸车购买价 | 从 `13.39万 裸车购买价` 提取，仅用于购车口碑 |
| 参考价格 | 从 `15.99万 参考价格` 提取，仅用于试驾探店口碑 |
| 购买时间 | 从 `2025-10 购买时间` 提取 |
| 探店时间 | 从 `2025-10 探店时间` 提取 |
| 购买地点 | 从 `赤峰 购买地点` 提取 |
| 探店地点 | 从 `芜湖 探店地点` 提取 |
| 评价详情 | 从详情页正文区域提取原始展示文本，不抓评论区、推荐区 |
| 最满意 | 从详情页 `最满意` / `好评` 标题对应正文提取 |
| 最不满意 | 从详情页 `最不满意` / `槽点` 标题对应正文提取 |
| 来源链接 | 提取“查看完整口碑”详情页 URL |
| 抓取页码 | 记录当前分页编号 |

### 5.1 用户名规范化规则

- 不删除“认证”信息
- 若页面展示为：`真实用户名 风云X3L认证`
- 输出规范化为：`真实用户名_风云X3L认证`
- 普通 `真实用户名 风云X3L` 形式可按需保留原样或仅去掉车型尾缀，但不得改动用户名本体

### 5.2 数据类型判定规则

满足任一条件时，判为 `试驾探店口碑`：

- 出现 `参考价格`
- 出现 `探店时间`
- 出现 `探店地点`
- 正文标题为 `好评` 或 `槽点`
- 标题或正文明确出现 `试驾` 或 `探店`

否则默认判为 `车主购车口碑`。

## 6. 技术路线

### 6.1 主方案：列表页提链 + 详情页提文

- 使用 headless browser 打开全部口碑列表页
- 从列表页提取结构字段与详情链接
- 再进入详情页抓取正文与维度分数
- 优先读取页面文本和 DOM，不默认走 OCR

### 6.2 分页遍历

- 若用户明确指定页码范围，则按指定范围遍历
- 若用户未指定最后页，或表达“全部页面 / 所有页面 / 所有条目”，则先自动检测分页总页数
- 自动检测可通过首页分页控件、尾页链接或分页文本（如“共 N 页”）取得
- 每页抓取完成后执行记录数与核心字段校验

---

## 7. 校验规则

- 每条记录至少应包含“用户名 / 综合口碑 / 车型 / 来源链接 / 评价详情”
- 缺失则标记为异常记录，不直接作为正式结果写入
- 导出后必须生成同名 `.validation.json`
- 若使用 `--strict-validate`，发现异常记录时返回非零退出码
