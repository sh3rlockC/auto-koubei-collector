# auto-koubei-collector

汽车之家车型口碑采集 skill。

用于从汽车之家口碑页批量采集指定车型的“最满意 / 最不满意”评价，并导出为**按来源链接严格对齐**的结构化 Excel。

## 当前正式输出

- 只导出 **2 个 sheet**：
  - `购车口碑`
  - `试驾口碑`
- 每一行代表 **同一个来源链接** 的同一条口碑
- 同时包含两列：
  - `最满意`
  - `最不满意`
- 仅当同一来源链接下 **最满意 / 最不满意都有内容** 时，才进入正式结果
- 导出后自动生成同名 `.validation.json` 校验报告
- 支持 `--strict-validate`，校验失败时直接返回非零退出码

## 功能

- 支持汽车之家车型口碑页抓取
- 支持 `最满意` / `最不满意` 两个维度
- 支持分页批量抓取
- 按来源链接对齐同一条口碑
- 区分：
  - 车主购车口碑
  - 试驾探店口碑
- 导出为双 sheet 对齐版 Excel
- 导出后自动生成同名 `.validation.json` 校验报告
- 支持文件命名规则：
  - `ZJ+车型名称+口碑对齐版_日期.xlsx`

## 校验逻辑

校验报告会包含：

- 原始抓取条数（最满意 / 最不满意）
- 对齐后正式结果条数
- 两个 sheet 的数量
- 每页解析到的卡片数量
- 未对齐链接清单
- 异常记录清单

## 迁移说明（旧版 → 新版）

### 旧版

- 4 个 sheet：
  - `最满意_车主购车口碑`
  - `最满意_试驾探店口碑`
  - `最不满意_车主购车口碑`
  - `最不满意_试驾探店口碑`
- 适合分别看两个维度
- 但当两边链接集合不完全一致时，不方便一一对齐

### 新版

- 2 个 sheet：
  - `购车口碑`
  - `试驾口碑`
- 同一来源链接一行展示
- 同时包含：
  - `最满意`
  - `最不满意`
- 更适合做对照分析、摘要归纳、情感对比

### 行为变化

- 旧版：允许最满意 / 最不满意分别独立导出
- 新版：必须按来源链接对齐，缺任一侧就不进入正式结果
- 旧版校验：偏重总数 / 分页一致性
- 新版校验：增加未对齐链接、缺正文异常、原始解析异常
