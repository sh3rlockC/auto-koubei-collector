# 汽车之家车型口碑采集

按下面流程执行，目标是稳定抓取汽车之家车型口碑，并输出可复核、可分析的 **来源链接对齐版 Excel**。

## 1. 确认输入

优先接受三种输入：

- 完整口碑页 URL
- `seriesId + 页码范围`
- 自然语言任务描述（如“抓风云X3L最满意和最不满意 1-13 页，按来源链接对齐导出 Excel”）

已确认的维度规则：

- `dimensionid=10` → 最满意
- `dimensionid=11` → 最不满意

分页规则：

- 第 1 页：`https://k.autohome.com.cn/{seriesId}?dimensionid={dimensionid}&order=0&yearid=0#listcontainer`
- 第 n 页：`https://k.autohome.com.cn/{seriesId}/index_{n}.html?dimensionid={dimensionid}&order=0&yearid=0#listcontainer`

## 2. 抓取方式

优先使用浏览器方式读取页面可见文本，不要默认走 OCR。

执行原则：

- 先打开列表页
- 从列表页提取结构字段
- 按顶层口碑卡片切分，不要按内部 `listitem` 粗暴切
- 从每条卡片里提取详情链接
- 最终按来源链接对齐最满意 / 最不满意

## 3. 当前正式输出结构

当前正式版输出 **2 个 sheet**：

- `购车口碑`
- `试驾口碑`

每行字段为：

1. 数据类型
2. 用户名
3. 发表日期
4. 口碑标题
5. 综合口碑
6. 车型
7. 行驶里程
8. 电耗
9. 裸车购买价
10. 参考价格
11. 购买时间
12. 探店时间
13. 购买地点
14. 探店地点
15. 最满意
16. 最不满意
17. 来源链接
18. 最满意抓取页码
19. 最不满意抓取页码

关键要求：

- 同一行必须是 **同一个来源链接**
- `最满意` 和 `最不满意` 两列都必须有内容
- 缺任一侧时，不写入正式结果，进入异常清单

## 4. 数据类型判定

满足任一条件时，判为 `试驾探店口碑`：

- 出现 `参考价格`
- 出现 `探店时间`
- 出现 `探店地点`
- 正文前缀为 `好评` 或 `槽点`
- 标题或正文明确出现 `试驾` 或 `探店`

否则默认判为 `车主购车口碑`。

当最满意 / 最不满意两侧类型判定不一致时：

- 只要任一侧满足试驾条件
- 则该来源链接整条记录统一归为 `试驾探店口碑`

## 5. 校验规则

正式导出后，必须生成同名 `.validation.json`，至少包含：

- 最满意原始抓取条数
- 最不满意原始抓取条数
- 对齐后正式结果条数
- `购车口碑` / `试驾口碑` 两个 sheet 数量
- 每页解析到的卡片数量
- 未对齐来源链接清单
- 异常记录清单

处理原则：

- 只要某个来源链接无法同时拿到最满意 / 最不满意两列，就不得写入正式 Excel
- 若启用 `--strict-validate`，存在异常时返回非零退出码

## 6. 附带脚本

- `scripts/export_autohome_koubei.py`

示例：

```bash
python3 skills/auto-koubei-collector/scripts/export_autohome_koubei.py \
  --series-id 8208 \
  --start-page 1 \
  --end-page 13 \
  --output ./ZJ风云X3L口碑对齐版_2026-03-26.xlsx \
  --workdir /Users/xyc/.openclaw/workspace
```
