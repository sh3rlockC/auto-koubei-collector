# 汽车之家车型口碑采集

按下面流程执行，目标是稳定抓取汽车之家车型口碑，并输出可复核、可分析的 Excel。

## 1. 确认输入

优先接受三种输入：

- 完整口碑页 URL
- `seriesId + 页码范围`
- 自然语言任务描述

当前抓取模型默认走“全部口碑”列表：

- 第 1 页：`https://k.autohome.com.cn/{seriesId}?dimensionid=0&order=0&yearid=0#listcontainer`
- 第 n 页：`https://k.autohome.com.cn/{seriesId}/index_{n}.html?dimensionid=0&order=0&yearid=0#listcontainer`

页码处理规则：

- 如果用户明确给了页码范围，就按用户指定页码抓取
- 如果用户没有给页码，或明确说“全部页面 / 所有页面 / 所有条目”，则先检测该车型口碑分页总页数，再按 `1 ~ 最后一页` 抓取
- 若用户给了 `start-page` 但没给 `end-page`，脚本默认会自动探测 `end-page`

## 2. 抓取方式

优先使用浏览器方式读取页面可见文本，不要默认走 OCR。

执行原则：

- 先打开全部口碑列表页
- 从列表页提取结构字段与详情链接
- 再进入详情页抓取正文全文
- `评价详情` 只保留口碑正文，不抓评论区
- `最满意` / `最不满意` 从详情页正文中拆出，不再依赖列表页短摘要
- `综合口碑` 优先用详情页 7 个维度分数求平均；若详情页分数不完整，再回退到列表页原值

## 3. 字段清单

输出以下 19 个字段：

1. 数据类型
2. 用户名
3. 发表日期
4. 口碑标题
5. 综合口碑
6. 车型
7. 行驶里程
8. 电耗
9. 裸车购买价
10. 参考价格
11. 购买时间
12. 探店时间
13. 购买地点
14. 探店地点
15. 评价详情
16. 最满意
17. 最不满意
18. 来源链接
19. 抓取页码

字段原则：

- `评价详情` 必须保留网页原文，不删改、不摘要、不纠错
- `评价详情` 位于 `最满意` / `最不满意` 两列之前
- `最满意` / `最不满意` 必须从 `评价详情` 对应正文里拆出
- `综合口碑` 以详情页 7 维均值为主，保留两位小数
- `电耗` 保留页面原文，如“14.6kWh 春秋电耗”
- 页面无字段时留空，不强行补值
- 价格 / 时间 / 地点必须拆列，不再混到同一列

## 4. 用户名规范化

不要删除“认证”信息。

如果页面展示为：
- `真实用户名 风云X3L认证`

则规范化为：
- `真实用户名_风云X3L认证`

普通 `真实用户名 风云X3L` 形式：
- 可保留原样，或仅去掉车型尾缀
- 不得改动用户名本体

## 5. 数据类型判定

同时兼容两类记录：

- `车主购车口碑`
- `试驾探店口碑`

满足任一条件时，判为 `试驾探店口碑`：

- 出现 `参考价格`
- 出现 `探店时间`
- 出现 `探店地点`
- 详情页正文标题为 `好评` 或 `槽点`
- 标题或正文明确出现 `试驾` 或 `探店`

否则默认判为 `车主购车口碑`。

不要丢弃试驾/探店数据，除非用户明确要求只保留真实购车口碑。

## 6. 导出规则

默认文件命名：

- `ZJ+车型名称+口碑_页数范围.xlsx`
- 例如：`ZJ启源A06口碑_1-13页.xlsx`
- 若未显式指定页数范围，可省略页数后缀

当前脚本的实际输出是单 sheet 结构：

- `口碑`

每行保留一条完整口碑，并包含：

- `评价详情`
- `最满意`
- `最不满意`
- `来源链接`
- `抓取页码`

## 7. 校验规则

每条记录至少应包含以下核心字段：

- 用户名
- 综合口碑
- 车型
- 来源链接
- 评价详情

缺任意一项时：

- 标记为异常记录
- 不直接写入正式结果
- 记录到异常清单或 JSON 侧产物中

每页抓取后检查：

- 是否成功拿到口碑列表区
- 是否识别到合理数量的口碑卡片
- 是否出现大面积字段为空

正式导出后，必须再做一层结果级校验：

- 生成同名 `.validation.json` 报告
- 保留分页条数与异常原因

处理原则：

- 若只是普通导出：允许输出 Excel，但必须明确提示用户校验失败
- 若使用 `--strict-validate`：发现异常记录时返回非零退出码
- 发现类似“某一页整页漏写”这类问题时，要优先提示用户复跑，不要默认认为站点真实数据就是不一致

## 8. 重试规则

分页抓取失败时：

- 自动重试 2 次
- 连续失败后停止静默继续，明确报告页码和错误原因

常见瞬时错误包括：

- `ERR_ABORTED`
- 页面导航中断
- 页面内容未加载完整

## 9. 实操提醒

- 不要按卡片内部的子 `listitem` 粗暴切分记录
- 必须按顶层口碑卡片切分
- 不要因正文口语化、像广告、像生成式措辞就擅自删除，只处理明显不属于正文区域的解析噪音
- 评论区、同车系推荐、相关车系推荐不纳入 `评价详情`
- 批量任务除 Excel 外，最好同步保留 JSON 原始结构化结果，方便回溯和修规则

以下字段暂不纳入当前版本：

- 点赞数 / 评论数 / 浏览数
- 油耗 / 续航

## 12. 附带脚本

本 skill 附带脚本：

- `scripts/export_autohome_koubei.py`

用途：

- 按字段清单导出 Excel
- 默认按全部口碑分页抓取
- 从详情页抽取 `评价详情`、`最满意`、`最不满意`
- `综合口碑` 优先用详情页 7 维均值计算
- 导出后自动生成同名 `.validation.json` 校验报告

## 13. 进度输出

脚本支持三类进度输出：

- `--progress`
- `--progress-file /path/to/job.progress.json`
- `--progress-webhook https://...`
- `--feishu-webhook https://...`

采集层的进度粒度按阶段和页数计算：

- 自动探测总页数
- 采集全部口碑
- 写出 Excel
- 写出校验报告
